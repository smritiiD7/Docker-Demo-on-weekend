Docker CMD Instruction

The CMD instruction in a Dockerfile specifies the default command to run when a container starts. It is used to provide defaults for an executing container.

Example:

CMD ["python", "app.py"]

This means that when the container runs, it will execute python app.py unless a different command is provided at runtime.

Another Common Example:

CMD ["nginx", "-g", "daemon off;"]

This tells Nginx to run in the foreground (not as a background daemon), which is required to keep the Docker container running.
